4. Email Notifications - Detailed Breakdown

1. Survey Submission Notification

Source File: backend/emailService.js → sendSurveyNotification()
Trigger: Employee submits skill survey (POST /api/surveys)
Behavior: Fire-and-forget (.catch(), does NOT await delivery)

TOazka.innovation@gmail.com Hardcoded
CCNone
SubjectNew Survey Submission - {fullName}
ContentName, email, HR code, department, title, project name
2. Admin Custom Email (Survey)

Source File: backend/routes.js → POST /api/surveys/send-email
Trigger: Admin sends a custom email from admin panel
Behavior: Awaited

TOCustom (from request body)
CCNone
SubjectCustom (from request body)
ContentCustom HTML (from request body)
3. Annual Vacation Submission

Source File: backend/vacationRoutes.js → sendVacationEmails()
Trigger: Employee submits or updates annual vacation plan (POST /api/vacations)
Behavior: Awaited (2 separate emails sent)

Email #1 - Employee Notification

TOEmployee's email
CCEMAIL_CC env + Employee's Reporter (per-employee)
Subject{New/Updated} Vacation: {name} ({dept}) - {days} days ({year})
ContentEmployee details, vacation days list, submission date

Email #2 - Admin Copy

TOEMAIL_USER (sender/admin account)
CCNone
SubjectSame as Email #1
ContentSame as Email #1
4. Vacation Reminder (Daily Cron)

Source File: backend/vacationRoutes.js → GET /api/vacations/reminders/check
Trigger: Vercel cron daily at 6:00 AM UTC (8:00 AM Egypt time)
Cron Config: backend/vercel.json → "0 6 * * *"
Milestones: 30 20 10 5 3 1 days before vacation
Behavior: Groups all matching days per employee into 1 email

TOmohamed.essa@azka.com.eg Hardcoded
CCEmployee's Reporter (per-employee)
SubjectVacation Reminder: {name} ({dept}) - {X} day(s) away
ContentEmployee profile (name, email, HR code, dept, title, experience, reporter), upcoming days table with color-coded urgency, full vacation plan for context
5. Service Request Submission (WFH / Urgent Vacation / Need Help)

5. Email Flow Summary

--- Employee Actions ---
Employee submits survey → azka.innovation@gmail.com
Employee submits vacation → Employee + CC(EMAIL_CC, Reporter) + Admin(EMAIL_USER)
Employee submits WFH → Employee + CC(EMAIL_CC, Reporter) + Admin(EMAIL_USER)
Employee submits Urgent Vacation → Employee + CC(EMAIL_CC, Reporter) + Admin(EMAIL_USER)
Employee submits Need Help → Employee + CC(EMAIL_CC, Reporter) + Admin(EMAIL_USER)

--- Admin Actions ---
Admin approves/rejects request → Employee + CC(EMAIL_CC, Reporter)
Admin sends custom email → Custom recipient

--- Automated (Cron) ---
Daily 8AM Egypt (vacation reminder) → mohamed.essa@azka.com.eg + CC(Reporter)

6. Important Notes

1. Reporter assignment is per-employee, not per-department. Each employee has their own reporter defined in the reporter field of the EMPLOYEES array.

2. The survey notification goes to azka.innovation@gmail.com (hardcoded), not the admin account. No reporter CC on surveys.

3. Vacation reminders go only to the manager (mohamed.essa@azka.com.eg, hardcoded), with the employee's reporter CC'd.

4. The EMAIL_CC env variable is used for service requests and vacations, but NOT for survey notifications or vacation reminders.

5. WFH requests allow editing (upsert); Urgent Vacation and Need Help block duplicates (409 error if pending exists).

6. Survey notification is fire-and-forget; all other emails are awaited before responding to the client.

7. Vercel cron jobs require the Pro plan. On the Hobby plan, an external cron service is needed.
